软件开发

PC端软件开发，首先想到的是Windows桌面应用程序，因为迄今为止，国内个人电脑使用Windows系统的用户还是最多的。 虽有很多开发人员吐槽微软不开源、不跨平台，使用的开发语言C#也不像Java那样无所不能，但是纵观几十年来微软做出的成就，不可否认， PC端软件GUI处理得最好的还是微软。本人开发PC端软件主要使用C#语言，但也不局限于此，Java、C++和C也是我的辅助开发语言， 针对不同需求使用不同的开发语言，既能快速开发项目，又能兼容各个平台，这才是客户最想要的。

移动端APP开发，首先想到的是基于Android的应用开发，因为国内安卓系统用户仍然是最多的，但是也不否认IOS、Google Chrome等系统的强大。 本人善用Java语言做安卓应用开发，加上安卓应用程序的前端界面与Windows上WPF应用程序有异曲同工之处，使得我能够快速实现PC端和移动端应用程序的开发。

嵌入式开发，学习软件开发之前，最理想的开发工作其实就是嵌入式开发，因为我想在生活中创造科技感，想为我自己打造专属的智能家居，但是很遗憾，到现在为止，单片机还没有学习。 如果有机会，我还是会花时间来弥补我的遗憾。所以对嵌入式感兴趣的我也非常乐意去尝试嵌入式开发。

网站开发

网站开发，首先想到的还是PHP，目前国内中小型网站，PHP开发比较快速，能够使用简单的框架就可以完成。 PHP不是我的主要Web开发语言，但是我使用起来还是比较方便、快捷。开发语言没有局限性，什么语言开发快速、稳定性强、维护方便就用什么开发语言， 针对不同情况和客户需求进行选择是一个开发人员最应该考虑的，前提是你要学会多种开发语言，并且熟悉掌握它们之间的优劣之处。

网站开发，还应该想到的是ASP和ASP.NET Web应用开发，尤其是大型网站使用ASP.NET MVC模式进行开发，达到了前后端分离，易于后期维护的效果。 当然这也是我的主要Web开发语言，因为项目经历多，所以熟悉。另外微软推出ASP.NET Core Web，跨平台部署之后更使得ASP.NET在网站开发中大展拳脚。 所以，大中型网站的开发，ASP.NET无疑是一个较好的选择。

网站开发，还不能忘记的是无所不能的Java，Java Web之所以这么长久存在是因为Java程序员真的太多，只要熟悉基本框架，Java程序员就可以很熟悉开发Web应用。 当然，Web开发的三大主流开发语言中，JSP虽然近年来渐渐被忽视，但是还是存在的。本人也开发过JSP网站，但是Java Web不是我最擅长的。

数据处理

很幸运，我的大学没有选中我报考的专业，而调剂到了信息与计算科学这个专业，因为我们正赶上人工智能和大数据这班列车。

东北电力大学的信息与计算科学这个专业，正好培养了我数学和计算机两方面的能力，通过数学建模我彻底明白了怎么将数学运用于生活，怎么利用计算机来解决生活中的实际问题。 人工智能无非也就是让计算机来实现人的思维，将人的思维用算法来描述，通过计算机编写程序来具体实现。 然而，这其中少不了的就是数据处理，任何实际问题都离不开数据， 数据量多到一定程度，我们叫它大数据。针对海量数据，我们怎么快速处理就是我们需要解决的关键问题。学过大数据处理得人肯定对Hadoop框架不陌生，这也是我接触大数据处理得第一个框架， 通过这个框架的学习和实践，我真正了解了大数据处理的基本流程。

任职经历

在其位，谋其政！只有处在那个职位才能真切地体会到那个职位该干些什么！

我的大学，就担任过三个职位，虽然不多，但担任每一个职位我都尽职尽责，每一个职位都让我受益良多。

文体委员

文体委员是我上大学担任的第一个职位，大家都知道在大学担任班委必须通过竞选，民主投票通过才能任职，若要连任必须保证学年综合测评排名前30%（各学校要求有所不同）。 正因为如此，我从一个胆怯、不敢上台发言的大一新生开始，勇敢地走上讲台参加竞选，成功入选之后，我考虑事情的思维从一个人发展到一个大集体，需要与其他班委相互协作 统一好班级相关事宜，我负责班级文体活动，与学生会体育部对接。担任职位后，我不仅要服务同学，还要努力提升自己的学习成绩和与同学之间的沟通能力，作为班级文体委员， 我应该为他们树立榜样，学习成绩保持专业前10%，体育运动类项目保持班级第一，同时还辅导同学加强体育锻炼。三年的时间过去了，我依旧是班级体委，依然为同学们服务，因为这是我的责任。

C#小组组长

担任C#小组组长，让我彻底走上了编程之路。软件研发中心C#小组，主要学习C#语言，运用C#语言开发基于.NET平台的Windows桌面应用程序和ASP.NET网站。 刚开始加入软件研发中心时，主要由上一届学长学姐给我们统一培训，通过一段时间的培训之后，考核通过的同学将能够继续留在软件研发中心，通过竞选 担任部门部长或小组组长的同学将成为核心成员，接着为下一届新生做贡献。

自从担任C#小组组长之后，我的大学生活变得格外忙碌。一方面，作为一组之长，技术必定是小组内学得最好的，不然怎么能带好下一届。于是软件基地工 作室成了我大学常去的地方，每天除了上课时间几乎都在工作室学习开发技术，晚上很晚了才回到寝室休息。另一方面，任职就意味着服务他人，C#小组 作为软件研发中心的一个重要分支，从上任的那一刻起C#小组的事就是我的事，C#小组发展好与不好责任在我头上了。招新之后的那一个学期是最忙碌的， 既要学好专业知识，又要配合上一级主管管理好C#小组，作为组长，每周要给新生授课讲解开发知识，还要不断学习新技术。一年的辛苦总算没白费， 通过一年的时间，培养出了三个非常满意的徒弟，自己也学到了不少开发技术。

技术主管

软件研发中心运行模式是在实践基地老师的指导下，学生自主管理。中心聘请在校学生担任中心的行政主管（CEO）、技术主管、培训主管、项目经理、外联主管以及各部门部长。 中心大小事务由各主管共同商议，最后行政主管统一执行。

在软件研发中心忙碌了一整年的我本不打算竞选主管，想好好放松放松，慢慢地享受一下大学生活。可是，担任组长的一年时间里，我已经慢慢习惯了忙碌的生活，习惯了服务同学， 习惯了运用自己的思维做事。于是，我又选择了竞选技术主管，继续走我的编程之路。

担任软件研发中心技术主管这一年，让我的思维提高到了一个新的高度，特别是与别人的沟通特别讲究，真正体会到了在其位谋其政的深刻意义。每次与院长以及各位基地指导教师 谈话都会有新的收获，另外还认识了从软件研发中心走出去的学长学姐，从学长学姐们的行动中我体会到了他们对软件研发中心深深的情谊，他们时刻关注着软件研发中心的发展。 毕业十年的学长学姐仍然放不下软件研发中心，可见这份情，是多么的深！

担任技术主管的一年里，我们几个主管主要讨论的是怎么让中心发展得更好，我们尝试过多种改革措施，虽然到我离职的时候还未见成效，并且我们也失败过多次，但是我们还是 收获了很多。软件研发中心，是探索技术的地方。所以最大的改革还是培训方案的改革，怎么让中心成员学到更多的新兴技术是我们最关切的，所以，我们在任时提出了并组办了 自学班，但是效果不是理想的。今年要离职之际，我提出了“引导式-探索式-讨论式”的培训方案，这将让下一届的同学来执行，我还期待着成效。然而，在我们努力学习旧知识 的同时，新兴的开发技术发展得飞快，我们都被落下了，所以今年我们又将中心的小组制改革为团队制，希望我们在学习新知识的同时加强团队建设，让我们软件研发中心这个大家 庭更有气氛。

我的项目开发经验主要是C#语言开发较多，完成过几个Windows桌面应用的开发，几个ASP.NET不同框架的网站开发，下面是我的一些项目摘要。

签到系统

2016年7月

2016年7月是我参加的第一个暑期培训，软件研发中心每年暑假都会组织中心成员集体培训，而培训方式主要通过实际项目开发来学习开发技术，从而同学们的实战能力。 暑假期间，我参与了软件研发中心签到系统的开发，是使用微软推出的WPF技术进行开发的Windows桌面应用程序，我负责数据库设计部分，使用的数据库是SQL Server 2012版本， 由于当时技术还不够，大部分时间花去补漏洞去了，数据库设计完成后，我花了整整两周时间看完“SQL Server 从入门到精通”一书，主要学到了数据库的基本操作(CRUD)， 数据库备份，创建触发器等技术。开学之后软件投入使用，数据库的维护工作就是我来完成。

新生考试系统

2017年3月

2017年3月，正直开学初，我灵感一来，想做一个考试软件，方便新生测试使用。于是我提前一周回学校开始了我的第一个完整项目的开发， 由于当时学的知识都比较旧，实现了基本功能，然而前端设计不是很可观没有投入使用。这个软件同样使用了WPF技术进行开发， 使用了SQL Server 2012版本的数据库来存储数据。主要完成的功能有，上传Word格式题目、考生通过学号登录，可以选择C语言、PS、Web进行作答， 规定时间内将文件保存并上传服务器，管理员登录可查看考生选择、填空成绩，手动录入程序题得分，并可以打印考生成绩表。

签到系统更新

2017年7月

软件研发中心签到系统运行一年之后，我们收到了中心成员的反馈信息，并发现了很多漏洞。刚投入使用的签到系统界面没有经过美工人员的设计，直接投入使用， 众多工作室成员提议界面不够友好，需要美化，另外，软件运行一段时间之后IP限制出现了问题，还有工作室成员反应打开WiFi之后，电脑无法进行签到。 2017年7月，又迎来了一个暑假，没有师傅得带领，我也成为了别人的师傅，组织小组成员对签到系统进行了更新，具体实现了使用样式表美化客户端界面， 增加小游戏界面，修复ip限制的bug，自定义安装向导。当然，这些功能的完善少不了学弟们的辛勤付出，我只是给他们答疑解惑罢了。

更新房产管理平台网站

2017年10月

2017年9月刚开学不久，学校房产管理处老师联系我们基地指导教师，需要我们为其更新一个2010级学长给学校做的一个房产管理网站，当时我正开始学习ASP.NET网站开发， 然而我的师傅刚好出去实习了，这个重任就交给我来完成了。

此网站是使用ASP.NET WebForm框架搭建的学校房产管理网站，主要用于对学校所有建筑、楼宇、房间进行管理，并能够线上实时查询、统计报表。由于开发时间较久远，浏览器更新较快，导致学长开发的 网站不再兼容新兴浏览器，导致登录权限发生错误，加上布局过于紧凑，网站看起来已经不如人眼。接手网站更新工作之后我又开始忙碌了，因为很多关于网站的开发技术 还不熟悉得现学现用。通过一个月的时间，我完成了网站的更新，并添加了在线预览CAD图纸功能，完善了数据库分类报表功能。

软件研发中心网站

2018年2月

2018年1月正是一个寒冷的冬天，我的寒假是裹着被子在凳子上敲代码过来的。当时了解了ASP.NET跨平台部署技术，就尝试着修改中心官方网站，想着将其部署到云服务器上以便于外界访问， 而不限于学校的局域网内。通过一个假期的挣扎，并没有实现最初的梦想，虽然自己购买了服务器，为中心买了域名。也学会了云服务器上部署ASP.NET网站，但是由于把大多时间用去学习云 端技术去了，没有完成中心网站的跨平台开发。

虽然没有完成开发，但也有了很多收获。此次开发使用了ASP.NET CORE MVC技术进行开发，数据库使用MYSQL5.5.8版本，并且使用EntityFramework技术来实现数据库设计，另外初次使用了 LINQ技术和lambda表达式。这也就是我未完成项目开发的原因。想用的技术都不熟悉，都需要先学会了再使用，而我现学现用就出现了很多问题，耽误了很多时间，最后没有达到想要的结果。 开学之后又有了其他项目，所以此网站的开发就这样被搁置了。但是项目源码保存在GitHub上，有时间可以继续开发。

“创友之家”网站

2018年4月

2018年3月初，新学期又开始了，然而想到自己2017年申请的吉林省创新创业训练项目“创友之家”网站还没有完成开发，于是开学又开始忙碌的生活了。

此网站是使用的前端技术包括Web基本开发技术HTML、CSS、JavaScript以及BootStrap框架。 后端技术包括ASP.NET网站开发技术、ASP.NET MVC开发模 式、LINQ语言集成查询技术、Lambda表达式、Entity Framework实体框架等。 服务器使用的是IIS服务器。多亏了寒假的挣扎，学到的技术感觉没那么陌生， 使用起来有了更深的理解，不再像之前那么迷茫。所以通过一个多月的时间，“创友之家”网站顺利完成，答辩时还得到了老师的一致好评。
